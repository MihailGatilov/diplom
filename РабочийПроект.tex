\section{Рабочий проект}
\subsection{Описание модулей разработанной программной системы}

Разработанная программно-информационная система регистрации данных пользователей реализована в виде одностраничного web-приложения (Single Page Application), построенного с использованием фреймворка Vue~3 и языка TypeScript. Архитектура приложения основана на доменном подходе, обеспечивающем разделение функциональности по предметным областям и снижение связанности между отдельными модулями системы.

Каждый домен включает компоненты пользовательского интерфейса, бизнес-логику, средства доступа к данным и описание используемых типов данных. Такой подход упрощает сопровождение и расширение системы, а также повышает удобство тестирования отдельных функциональных частей приложения.

Пример реализации метода получения списка заявок приведён ниже.

\begin{lstlisting}[language=Java]
	export const requestsApi = {
		async getAll(): Promise<ServiceRequest[]> {
			const { data, error } = await supabase
			.from('service_requests')
			.select('*')
			
			if (error) {
				throw error
			}
			
			return data
		}
	}
\end{lstlisting}

Использование единой точки доступа к данным упрощает модификацию приложения при изменении используемых внешних сервисов и облегчает проведение модульного тестирования.

Использование конечного автомата позволяет исключить возникновение некорректных состояний заявок и обеспечивает соблюдение регламентированных бизнес-процессов организации.

\subsection{Системное тестирование}

Системное тестирование проводилось путём проверки основных пользовательских сценариев работы программно-информационной системы через веб-интерфейс приложения. Целью тестирования являлось подтверждение корректности функционирования реализованных бизнес-процессов и соответствия системы требованиям технического задания.

\subsubsection{Тестовый сценарий: Авторизация пользователя}

\textbf{Предусловие:} пользователь зарегистрирован в системе и обладает действующей учётной записью.

\textbf{Тестовый случай:} пользователь вводит адрес электронной почты и пароль, после чего нажимает кнопку "<Войти">.

\textbf{Ожидаемый результат:} система выполняет проверку введённых данных посредством обращения к Supabase, формирует пользовательскую сессию и перенаправляет пользователя на главную панель приложения в соответствии с назначенной ролью.

\textbf{Результат:} тест пройден успешно.
